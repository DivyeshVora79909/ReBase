Schemaless: INSERT/CREATE without schema; schemafull: DEFINE TABLE/FIELD. CRUD: CREATE/INSERT, SELECT, UPDATE (SET/UNSET), DELETE (DELETE/REMOVE TABLE). Nested objects/arrays allowed; IDs are table:id. Record links: field holds record<type> (unidirectional), bidirectional with REFERENCE + COMPUTED <~ (v3+). Graph relations: RELATE source->edge->target, edge is table with in/out. Arrow syntax: -> out, <- in. Edges can have metadata (SET). Define table TYPE RELATION IN/OUT for type safety. Symmetric edges: sorted key field + UNIQUE index; traverse with <-> (both dirs). Recursive: @.{n} path repetition, recursive nesting with .@. Knowledge graph: typed entities, edges with metadata. Design: stable IDs, query planning for edge granularity. Prefer RL for simple links, ON DELETE control, performance; GR for complex multi-hop, relationship metadata, edge counts/weights. Edge removal when endpoint deleted; ENFORCED flag blocks edge before endpoints exist. Traversal direct from IDs: id->edge->target or destructuring. Flattening vs structured shape.
